# 건물 이펙트 애니메이션 설계

## 목적
쿠키 클리커에 생동감을 더하기 위해, 플레이어가 생산 건물을 보유하고 있는 동안 쿠키 주변에 해당 건물의 애니메이션 이펙트를 표시한다. 손수 굽기를 사면 파티시에가 반죽을 미는 모습, 작은 화덕을 사면 화덕에서 쿠키가 구워지는 모습(+연기), 쿠키 공장을 사면 공장이 가동되는 모습(+연기)이 보이도록 한다.

## 핵심 기능
- **표시 시점**: 구매 순간의 일회성 팝업이 아니라, 해당 건물을 1개 이상 보유하는 동안 계속 보이는 상태 표시. 건물을 팔거나 보유 개수가 0이 되면(현재 게임엔 판매 기능이 없으므로 사실상 "구매 전"에만 해당) 이펙트도 사라진다.
- **적용 범위**: 손수 굽기, 작은 화덕, 쿠키 공장 3종 모두에 이펙트를 만든다.
- **배치**: 메인 쿠키 캔버스를 세로로 확장(예: 200×200 → 200×280)하고, 쿠키는 위쪽 중앙에 그대로 두고 그 아래 가로로 나란히 3개의 고정 슬롯(손수 굽기/작은 화덕/쿠키 공장 순서)을 배치한다. 보유하지 않은 건물의 슬롯은 비워둔다.
- **애니메이션 복잡도**: 건물당 2~3프레임의 간단한 픽셀 그리드 루프. 약 300ms 간격으로 프레임 전환.
- **비주얼 스타일**: 기존과 동일하게 코드로 직접 그리는 픽셀 아트(Canvas, 그리드 배열 방식). 외부 이미지 에셋 없음.

## 기술 스택
기존과 동일 — Vite + 바닐라 JavaScript, HTML5 Canvas. 새 의존성 없음.

## 아키텍처
기존 아키텍처(단일 상태 객체 + 게임 루프)를 그대로 유지한다. 새 DOM 요소나 별도 캔버스를 추가하지 않고, 기존 메인 캔버스 하나에 쿠키와 건물 이펙트를 함께 그린다. 애니메이션 프레임 전환은 렌더 호출 빈도가 아니라 `performance.now()` 기준 실시간으로 계산해, 게임 루프 타이밍이 바뀌어도 애니메이션 속도가 일정하게 유지되도록 한다.

## 데이터 흐름
```
게임 루프 (100ms tick, main.js 변경 없음)
   → renderCookie(canvas, state) 호출
       → 캔버스 클리어
       → 쿠키 그리기 (기존과 동일)
       → BUILDINGS 순회하며:
            state.buildings[id] > 0 이면
              frameIndex = floor(performance.now() / 300) % 해당 건물 프레임 수
              drawBuildingEffect(ctx, id, frameIndex, 슬롯위치, pixelSize)
            0개면 해당 슬롯은 비워둠
```

## 파일 변경
- `src/pixelart.js`: 건물별 2~3프레임 픽셀 그리드/색상 데이터 추가, `drawBuildingEffect(ctx, buildingId, frameIndex, originX, originY, pixelSize)` 함수 추가. 알 수 없는 `buildingId`나 프레임 인덱스는 조용히 no-op 처리(기존 `drawBuildingIcon`의 방어 패턴과 동일).
- `src/render.js`: `renderCookie` 내부에 건물 이펙트 행을 그리는 로직 추가. 캔버스 크기 변경에 맞춰 쿠키 원점 좌표 계산도 조정.
- `index.html`: `#cookie-canvas`의 `width`/`height` 속성을 세로로 확장.
- `src/main.js`: 변경 없음 — 기존 `renderCookie(canvas, state)` 시그니처 그대로 재사용.

## 에러 처리
- `drawBuildingEffect`는 정의되지 않은 `buildingId`나 그리드 배열 범위를 벗어난 `frameIndex`에 대해 그리기를 생략하고 조용히 리턴한다(예외를 던지지 않음).

## 테스트
- `pixelart.js`의 `drawBuildingEffect`는 기존 방식대로 가짜 ctx 객체(`fillRect`/`fillStyle` 기록)를 사용한 Node `assert` 스크립트로 검증한다.
- `render.js`의 실제 렌더링 결과는 Playwright로 헤드리스 브라우저를 띄우고 `canvas.getContext('2d').getImageData(...)`로 픽셀 데이터를 직접 읽어, 의도한 그리드/좌표와 일치하는지 확인한다(기존 쿠키 렌더링 검증에서 이미 사용해 유효성이 확인된 방법). 스크린샷은 헤드리스 브라우저의 캔버스 합성 제약으로 신뢰할 수 없으므로 참고용으로만 사용하고, 픽셀 데이터 비교를 주된 검증 수단으로 삼는다.

## 범위 밖 (Out of scope)
- 건물 판매/철거 기능 (현재 게임엔 없음 — 있다면 이펙트도 사라져야 하지만 이번 스코프 아님)
- 건물별 다중 개체 표시 (예: 손수 굽기 3개를 샀다고 파티시에 3명이 나오는 것 — 개수와 무관하게 슬롯당 이펙트 1개만 표시)
- 사운드 이펙트
- 이펙트 슬롯의 반응형/모바일 레이아웃 최적화
