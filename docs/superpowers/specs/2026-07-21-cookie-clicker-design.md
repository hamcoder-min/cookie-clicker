# 쿠키 클리커 게임 설계

## 목적
바이브코딩 연습용 미니 프로젝트. 쿠키/음식 테마의 클리커(방치형) 게임을 픽셀 아트 스타일로 제작한다. 학습 목적이 우선이므로 범위는 최소한으로 유지한다.

## 핵심 기능
- **쿠키 카운터**: 화면 중앙에 큰 픽셀 쿠키를 표시. 클릭하면 쿠키 +1 (클릭 시 간단한 이펙트).
- **생산 건물 2~3종**: 예) 손수 굽기 → 작은 화덕 → 쿠키 공장. 구매하면 초당 자동으로 쿠키를 생산(cps 증가). 구매할 때마다 다음 구매 가격이 상승(예: 15% 증가).
- **상점 패널**: 각 건물의 현재 보유 개수, 다음 구매 가격, 초당 생산량을 표시. 쿠키가 가격보다 적으면 구매 버튼 비활성화.
- **자동저장**: localStorage에 1~2초 주기로 게임 상태 저장. 페이지 로드시 저장된 상태를 복원(없으면 초기값 사용).

## 비주얼 스타일
- 픽셀 아트, 코드로 직접 그림 (Canvas에 픽셀 그리드(0/1 배열)를 그리는 방식). 외부 이미지 에셋 없음.

## 기술 스택
- Vite + 바닐라 JavaScript (프레임워크 없음)

## 아키텍처
단일 상태 객체 + 게임 루프 방식을 사용한다. 클래스 기반이나 프레임워크 도입은 이 프로젝트 범위에서 과함.

```
사용자 입력 (클릭 / 구매 버튼)
   → state.js의 액션 함수 호출 (addCookie(), buyBuilding(id))
   → gameState 갱신

게임 루프 (setInterval, 100ms)
   → cps만큼 쿠키 증가
   → render() 호출해 화면 갱신
   → 1~2초 주기로 localStorage에 저장

페이지 로드시
   → localStorage에서 state 복원 (없으면 초기값)
```

## 파일 구조
```
cookie-clicker/
├── index.html
├── src/
│   ├── main.js       # 진입점, 게임 루프, 이벤트 리스너
│   ├── state.js       # gameState 정의 + 액션 함수 (addCookie, buyBuilding, save/load)
│   ├── render.js       # DOM/Canvas 업데이트
│   └── pixelart.js      # 픽셀 아트 그리기 함수 (쿠키, 건물 아이콘)
├── package.json
└── vite.config.js (필요시)
```

## 에러 처리
- localStorage 파싱 실패(손상된 데이터) 시 초기값으로 폴백.
- 건물 구매 시 쿠키가 가격보다 부족하면 구매 버튼을 비활성화해 잘못된 구매를 방지.

## 테스트
- 별도 자동화 테스트는 작성하지 않는다 (연습용 프로젝트 스코프).
- 수동 확인 시나리오: 클릭으로 쿠키 증가 확인 → 건물 구매 후 자동 생산 확인 → 새로고침 후 상태 유지 확인.

## 범위 밖 (Out of scope)
- 업그레이드/프레스티지 시스템
- 사운드/BGM
- 다중 언어 지원
- 모바일 반응형 최적화 (있으면 좋지만 필수 아님)
